Spotting O(n²) before it bites

Toward your mission: ship code that stays fast as the data grows.

By the end of this lesson you'll glance at a loop and know, in one breath, whether it will crawl when the input gets big, then what to reach for instead.

The shape of the problem

“Quadratic” means the work grows with the square of the input. Double the data and you don't double the work; you quadruple it. That is fine for ten items and catastrophic for ten thousand.

Spot it in code

The tell is a loop inside a loop over the same data. Here are two versions of the same task: “does this array contain duplicates?”

O(n²) · every pair

for (let i = 0; i < a.length; i++)
  for (let j = i+1; j < a.length; j++)
    if (a[i] === a[j]) return true;
return false;

O(n) · one pass

const seen = new Set();
for (const x of a) {
  if (seen.has(x)) return true;
  seen.add(x);
}
return false;
Heuristic Trading time for memory: a hash set remembers what you've seen so you never re-scan. Most O(n²) scans hide an O(n) solution behind a Set or a Map.
Read complexity, step by step

Work out the complexity of this snippet. Try each step before revealing it.

for (const row of grid)        // A
  for (const cell of row)      // B
    total += cell;             // C
1. Count the loops that scale with input. Two nested loops: A over n rows, B over m cells.
2. What does the inner body cost? Line C is O(1): one addition, independent of size.
3. Combine. O(n·m), or O(n²) for a square grid. Nested ≠ automatically n²; it's the product of what each loop scales over.

Why do we drop constants and lower-order terms?

Big-O describes growth, not exact counts. As n heads to infinity, 3n² + 5n + 7 is dominated entirely by the n² term; the 5n and 7 become rounding error. We care how the curve bends, not where it starts.

Two nested loops, each iterating over the same n-sized array. Time complexity?

Each element is paired with every element → n × n comparisons → O(n²).
