# Comparisons — Side-by-Side Visual Comparisons Workflow

> Split compositions that make contrasts visceral and immediate.
> The divide itself is part of the design — a visual argument in two halves.

---

## Brand Identity

| Token | Hex | Role |
|---|---|---|
| Warm Cream | `#F6F3EB` | Shared background, neutral zone |
| Coral/Orange | `#D67056` | **Side A** accent — warm, human, before, old, emotional |
| Tan/Sand | `#E6CCAB` | Soft transitions, divider zone |
| Teal | `#70B8AD` | **Side B** accent — cool, technical, after, new, rational |
| Dark Charcoal | `#222222` | Labels, borders, shared structural elements |
| Muted Blue | `#5D95C6` | Neutral annotations, captions |
| Dark Grey Blue | `#2C313A` | Deep contrast areas, dramatic shadows |
| Mustard Yellow | `#EAB64D` | Key difference markers, "notice this" highlights |

**Font:** Clean monospace technical font like JetBrains Mono (JetBrainsMono Nerd Font)

---

## Overview

Comparison images place two things side by side to make differences and similarities visible at a glance. The split composition uses Coral for one side and Teal for the other, creating an immediate visual separation that maps to the conceptual contrast.

The key principle: **mirror structure, divergent content**. Both sides share the same spatial framework but differ in the details. This makes the comparison fair and the differences stark.

---

## Split Composition Formats

### Format 1: Vertical Split (Left / Right)

```
┌────────────┬────────────┐
│            │            │
│   SIDE A   │   SIDE B   │
│  (Coral)   │  (Teal)    │
│            │            │
│            │            │
└────────────┴────────────┘
```

**Aspect ratio:** 16:9 or 2:1
**Best for:** Before/after, old/new, approach A vs B, technology comparisons

### Format 2: Horizontal Split (Top / Bottom)

```
┌─────────────────────────┐
│         SIDE A          │
│        (Coral)          │
├─────────────────────────┤
│         SIDE B          │
│        (Teal)           │
└─────────────────────────┘
```

**Aspect ratio:** 9:16 or 1:2 (portrait) or 4:3 (landscape with narrow split)
**Best for:** Timeline comparisons, process evolution, layered concepts

### Format 3: Diagonal Split

```
┌─────────────────────────┐
│ ╲  SIDE A               │
│   ╲  (Coral)            │
│     ╲                   │
│       ╲                 │
│         ╲   SIDE B      │
│           ╲  (Teal)     │
└─────────────────────────┘
```

**Aspect ratio:** 16:9
**Best for:** Dynamic comparisons, energy-charged contrasts, dramatic reveals

### Format 4: Center Divide with Shared Element

```
┌────────────┬────────────┐
│            │            │
│   SIDE A   │   SIDE B   │
│        ┌───┴───┐        │
│        │SHARED │        │
│        └───┬───┘        │
│            │            │
└────────────┴────────────┘
```

**Aspect ratio:** 16:9 or 1:1
**Best for:** Comparisons with a common element (same person, same tool, same concept expressed differently)

## Divider Styles

The divider between sides is a design element, not just a border.

| Divider Style | Visual | Best For |
|---|---|---|
| **Hard Line** | Single Dark Charcoal (#222222) line, 2-3px | Clean, factual comparisons |
| **Torn Edge** | Rough, hand-drawn tear effect | Before/after, destruction/creation |
| **Gradient Blend** | Tan/Sand (#E6CCAB) gradient zone between sides | Subtle contrasts, spectrums |
| **Lightning Crack** | Jagged diagonal line with Mustard (#EAB64D) accent | Conflict, opposition, debate |
| **Shared Object** | An object straddles the divide | Same-thing-different-context comparisons |
| **Fade** | Side A fades out, Side B fades in | Evolution, transformation |
| **None** | White/cream gap between sides | Minimal, clean, let content speak |

## Mirror Structure System

Both sides must share structural elements to make the comparison fair:

### Structural Mirrors

| Element | Side A | Side B |
|---|---|---|
| **Subject position** | Same placement | Same placement |
| **Scale** | Same size | Same size |
| **Camera angle** | Same perspective | Same perspective |
| **Background complexity** | Same level | Same level |
| **Label position** | Same location | Same location |

### Divergent Elements

| Element | Side A (Coral) | Side B (Teal) |
|---|---|---|
| **Color accent** | Coral (#D67056) washes/highlights | Teal (#70B8AD) washes/highlights |
| **Mood** | Varies (warm, old, human, emotional) | Varies (cool, new, technical, rational) |
| **Detail content** | Specific to concept A | Specific to concept B |
| **Texture** | Can vary | Can vary |
| **Typography tone** | Matches side A concept | Matches side B concept |

## Workflow Steps

### Step 1: DEFINE THE CONTRAST

**Goal:** Clearly articulate what is being compared and why.

Answer these questions:
1. **What is Side A?** (name, concept, technology, approach)
2. **What is Side B?** (name, concept, technology, approach)
3. **What is the relationship?** Choose one:
   - **Before / After** — temporal change
   - **Old / New** — version or generation comparison
   - **Approach A / Approach B** — methodology comparison
   - **Problem / Solution** — showing the fix
   - **Theory / Practice** — ideal vs. reality
   - **Simple / Complex** — scale comparison
   - **Human / Machine** — organic vs. synthetic
   - **Good / Bad** — quality comparison (be opinionated)
4. **What is the key insight?** What should the viewer conclude from the comparison?
5. **Is there a preferred winner?** Or is it a neutral presentation?

**Color assignment:**
- Side A → Coral (#D67056) — typically the warmer, more human, or "before" side
- Side B → Teal (#70B8AD) — typically the cooler, more technical, or "after" side

If the comparison is neutral, the assignment is arbitrary but must be consistent.

**Output:** Contrast brief with color assignments and key insight.

---

### Step 2: CHOOSE FORMAT AND DIVIDER

**Goal:** Select the split format and divider style.

Decision matrix:

| Comparison Type | Recommended Format | Recommended Divider |
|---|---|---|
| Before / After | Vertical Split (L/R) | Torn Edge or Hard Line |
| Old / New | Vertical Split (L/R) | Fade |
| Approach A vs B | Vertical Split (L/R) | Hard Line or None |
| Problem / Solution | Vertical Split (L/R) | Lightning Crack |
| Theory / Practice | Horizontal Split (T/B) | Gradient Blend |
| Simple / Complex | Vertical Split (L/R) | Hard Line |
| Timeline | Horizontal Split (T/B) | Gradient Blend |
| Dramatic contrast | Diagonal Split | Lightning Crack |
| Shared element | Center Divide | Shared Object |

Also determine:
- **Aspect ratio** based on format
- **Label placement** — top of each side, integrated into design, or bottom caption
- **Annotation need** — are Mustard (#EAB64D) highlight markers needed for key differences?

**Output:** Format, divider, aspect ratio, and annotation plan.

---

### Step 3: DESIGN MIRROR STRUCTURE

**Goal:** Define the shared visual framework that both sides will use.

Create a structural template:

```
Shared Framework:
  Layout: [vertical/horizontal/diagonal split]
  Subject type: [person, object, diagram, scene, code, interface]
  Subject position: [center of each half, offset, etc.]
  Subject scale: [percentage of half-width]
  Background: [minimal, environmental, abstract]
  Label style: [top header, integrated, caption]

Side A (Coral):
  Subject variant: [specific description]
  Coral accent applied to: [what elements]
  Supporting details: [environment, objects, text]
  Emotional tone: [warm, nostalgic, problematic, etc.]

Side B (Teal):
  Subject variant: [specific description]
  Teal accent applied to: [what elements]
  Supporting details: [environment, objects, text]
  Emotional tone: [cool, modern, solved, etc.]

Key Difference Markers:
  [List 2-3 specific visual differences to highlight with Mustard (#EAB64D)]
```

**Output:** Detailed structural template.

---

### Step 4: CRAFT THE PROMPT

**Goal:** Build the generation prompt.

#### Prompt Template

```
Side-by-side comparison illustration on warm cream (#F6F3EB) background.

[FORMAT]: [Left/Right OR Top/Bottom OR Diagonal] split composition.

Shared structure: Both sides show [SUBJECT TYPE] at [POSITION], same scale, same perspective. Charcoal sketch technique with selective watercolor accents. Clean monospace technical font like JetBrains Mono for all labels and text.

LEFT SIDE / SIDE A — "[LABEL A]":
[DETAILED DESCRIPTION OF SIDE A CONTENT]
Coral/orange (#D67056) watercolor wash accents on [SPECIFIC ELEMENTS].
Mood: [EMOTIONAL TONE]

RIGHT SIDE / SIDE B — "[LABEL B]":
[DETAILED DESCRIPTION OF SIDE B CONTENT]
Teal (#70B8AD) watercolor wash accents on [SPECIFIC ELEMENTS].
Mood: [EMOTIONAL TONE]

Divider: [DIVIDER STYLE DESCRIPTION] between the two sides.

Key difference markers: Mustard yellow (#EAB64D) small highlights on [2-3 SPECIFIC DIFFERENCES].

Labels "[LABEL A]" and "[LABEL B]" in dark charcoal (#222222) at the [top/bottom] of each side.

Overall style: Editorial illustration, charcoal sketch base with selective color. Both sides must maintain mirror structure — same spatial framework, different content. The comparison should be immediately readable at a glance.

Both coral (#D67056) and teal (#70B8AD) must be prominently visible.
```

### Step 7: VALIDATE

**Goal:** Verify the comparison meets all standards.

#### Validation Checklist

**Mirror Structure:**
- [ ] Both sides share the same spatial framework
- [ ] Subject placement is consistent between sides
- [ ] Scale is equal on both sides
- [ ] Neither side is visually dominant over the other (unless intentional)
- [ ] The comparison is readable at a glance — you immediately see "two things being compared"

**Color Compliance:**
- [ ] Side A uses Coral (#D67056) as its primary accent
- [ ] Side B uses Teal (#70B8AD) as its primary accent
- [ ] Background is Warm Cream (#F6F3EB)
- [ ] Mustard Yellow (#EAB64D) marks key differences (if applicable)
- [ ] Dark Charcoal (#222222) used for labels and structural elements
- [ ] Colors are NOT mixed between sides (Coral stays on A, Teal on B)

**Composition:**
- [ ] Split format matches the comparison type
- [ ] Divider style is appropriate and visible
- [ ] Labels clearly identify each side
- [ ] Negative space is balanced between sides
- [ ] The divider doesn't obscure important content

**Content:**
- [ ] Both sides accurately represent their respective concepts
- [ ] Key differences are visually apparent
- [ ] The intended insight/conclusion is supported by the visual
- [ ] No misleading visual bias (unless the comparison is intentionally opinionated)

**Typography:**
- [ ] Labels are legible and use monospace styling
- [ ] Text is consistent in size and placement on both sides
- [ ] Any annotations are in the correct brand color

**Technical:**
- [ ] Web-optimized version created
- [ ] WebP conversion completed
- [ ] Thumbnail created
- [ ] All files in `~/prj/art-skill/outputs/`

#### Failure Protocol

| Issue | Fix |
|---|---|
| One side dominates visually | Reduce detail/color intensity on dominant side, increase on weak side |
| Mirror structure broken | Regenerate with more explicit spatial constraints in prompt |
| Colors bleed between sides | Add explicit "Side A uses ONLY coral, Side B uses ONLY teal" to prompt |
| Comparison unclear | Add labels, simplify content, or switch to a clearer format |
| Divider too subtle | Switch to a stronger divider style (Hard Line or Lightning Crack) |

## Examples

### Example 1: Before/After Code Refactoring

**Contrast:** Messy code (before) vs. Clean code (after)
**Format:** Vertical Split
**Divider:** Hard Line

**Prompt:**
```
Side-by-side comparison illustration on warm cream (#F6F3EB) background.

Vertical split composition, left and right halves.

Shared structure: Both sides show a code editor / text block rendered in charcoal sketch style. Same scale, same perspective — a slightly angled view of a code document. Clean monospace technical font like JetBrains Mono.

LEFT SIDE — "BEFORE":
Dense, tangled charcoal lines suggesting messy code. Lines overlap, cross each other. A function that sprawls vertically. Comments scattered randomly. Arrows and dependencies creating a web. The feeling of chaos despite structure.
Coral/orange (#D67056) watercolor wash over the tangled areas — warm but messy, human but uncontrolled.
Mood: Overwhelming, cluttered.

RIGHT SIDE — "AFTER":
Clean, organized charcoal lines suggesting refactored code. Clear sections with whitespace between them. Short functions stacked neatly. Consistent indentation visible even in sketch form. The feeling of order and intention.
Teal (#70B8AD) watercolor wash over the clean structure — cool, precise, intentional.
Mood: Calm, controlled, satisfying.

Divider: Single dark charcoal (#222222) vertical line, clean and straight.

Key differences: Mustard yellow (#EAB64D) small arrows pointing to: (1) the sprawling function vs. short functions, (2) tangled arrows vs. clean hierarchy.

Labels "BEFORE" and "AFTER" in dark charcoal (#222222) at top of each side.

Both coral and teal prominently visible.
```

### Example 2: Human vs. AI Decision Making

**Contrast:** Human intuition vs. AI logic
**Format:** Center Divide with Shared Element
**Divider:** Shared Object (a decision/choice point in the center)

**Prompt:**
```
Side-by-side comparison on warm cream (#F6F3EB) background.

Vertical split with a shared central element. A crossroads/fork in the road sits at the center divide, straddling both sides.

LEFT SIDE — "INTUITION":
An angular figure (planeform style) with a warm glow from the chest area. Thought bubbles contain fragments — faces, memories, feelings rendered as loose charcoal sketches. The path on this side curves and meanders organically.
Coral (#D67056) watercolor wash on the figure and their thought fragments.

RIGHT SIDE — "LOGIC":
An angular figure with a grid/circuit pattern visible in the head area. Thought bubbles contain structured elements — flowcharts, data points, probability numbers. The path is straight and gridded.
Teal (#70B8AD) watercolor wash on the figure and data visualizations.

CENTER: The crossroads where both paths meet. Both figures face the same choice. Mustard yellow (#EAB64D) highlight on the shared decision point.

Dark charcoal (#222222) line work throughout. Labels in monospace font. The question is not which is better — both are valid paths.

Both coral and teal prominently visible.
```

### Example 3: Simple vs. Complex Architecture

**Contrast:** Simple monolith vs. Complex microservices
**Format:** Vertical Split
**Divider:** Gradient Blend (Tan/Sand)

**Prompt:**
```
Side-by-side comparison on warm cream (#F6F3EB) background.

Vertical split with a tan/sand (#E6CCAB) gradient blend zone in the center.

LEFT SIDE — "SIMPLE":
A single, solid architectural block rendered in charcoal. One building, one door, one window. Clean, unadorned. A single arrow going in, a single arrow coming out. Peaceful, quiet, boring.
Coral (#D67056) watercolor wash on the solid block — warm, reliable, unglamorous.

RIGHT SIDE — "COMPLEX":
Dozens of tiny architectural blocks connected by an intricate web of arrows, pipes, and bridges. Each block is individually clean, but the connections create visual chaos. Monitoring dashboards, log streams, service meshes rendered as tangled sketches.
Teal (#70B8AD) watercolor wash on the network of connections — cool, sophisticated, overwhelming.

The gradient blend zone shows the transition: one block beginning to fragment into many.

Mustard yellow (#EAB64D) on the total arrow count: one on the left, dozens on the right.

Dark charcoal (#222222) line work. Monospace labels. The irony should be visible: simplicity moved, not eliminated.

Both coral and teal prominently visible.
```
